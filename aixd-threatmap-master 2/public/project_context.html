AIxD Threat Map

Project Context & Developer Guide

Purpose

Filterable, searchable database of ~190 AI threats and opportunities to democracy, mapped to the IDEA Democracy Assessment Framework (P4Democracy aspect codes). Built for P4D researchers to share with policymakers. The map is an embeddable static component — no server, no database, no backend. All interactivity is client-side React. Data is compiled from CSV at build time.

Setup

# 1. Navigate to the source directory
cd "aixd-threatmap-master 2"

# 2. Install JavaScript dependencies
bun install

# 3. (One-time) Set up Python preprocessing
cd preprocessing
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cd ..

# 4. Run dev server (http://localhost:4321)
bun run dev

Deployment

GitHub Pages

Push to dev or main triggers the GitHub Action, which:

  1. Installs Bun + dependencies
  2. Runs bun run build
  3. Uploads dist/ as artifact
  4. Deploys to GitHub Pages

Embedding in P4D website

The map is fully static and can be embedded anywhere:

  • Iframe: <iframe src="https://org.github.io/ai-democracy-map/">
  • Static files: Copy dist/ contents to any server

If deployed under a subdirectory, set these env vars at build time:

Architecture

  • Single page: One Astro page renders <ThreatMap client:load />
  • All client-side: React handles filtering, searching, sorting, infinite scroll
  • Runtime data: useDataLoader fetches data.json + aspects.json via HTTP
  • No server: No SSR, no API routes, no database, no auth
  • Git workflow: CSV changes → commit → auto-rebuild → deploy

For the full architecture (component map, design system, conventions, infinite scroll, sticky filterbar, URL state), see CLAUDE.md.

Contribution Guide

For developers

  1. Fork or clone from P4D org
  2. Follow Quick Start to get running locally
  3. Edit components in src/components/
  4. Test: bun run dev (hot reload at localhost:4321)
  5. Build check: bun run build (must pass)
  6. Commit: imperative, granular, no co-author tags
  7. Push to dev → GitHub Action deploys preview

For researchers (data updates)

  1. Edit data/raw/mapping.csv
  2. Run bun run preprocess to regenerate data.json
  3. Commit both files, push → auto-deploy

Code conventions

  • Single React island — all interactivity inside ThreatMap
  • sourceShort in table, full source in expanded row
  • No overflow-hidden on table cells
  • Brand colors carry semantic meaning

Initial Setup & Verification 2026-06-10

Objective: Get the project running locally from a fresh clone.

Environment: macOS 12, Node.js v20.16.0 → upgraded to v22.22.3, Bun 1.3.14, Python 3.14.4

Issues:

  • Node.js too old (v20). Astro 6 requires >= 22.12.0. Fix: nvm was installed but not in default shell PATH. Sourced it and ran nvm use 22. Node v22.22.3 was already installed under nvm.
  • Python venv broken by $ in path. The repo path Billion$Connections/ contains a literal dollar sign. When python3 -m venv creates scripts, $Connections is interpreted as shell variable expansion at runtime, breaking pip, pytest, etc. Fix: Created venv at /tmp/aixd-venv, copied to preprocessing/.venv, then patched all scripts in .venv/bin/ to escape $Connections as \$Connections.
