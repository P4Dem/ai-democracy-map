# Researcher Guide — Updating the AI x Democracy Threat Map

This guide is for P4D researchers who need to update the data, the democracy framework codebook, or the website content. No programming experience is assumed.

---

## What this project is

The AI x Democracy Threat Map is a website that visualises P4D's research database — threats, solutions, and opportunities mapped to the IDEA Democracy Assessment Framework. It lives at a public URL and is updated by pushing changes to a GitHub repository.

**Key files you'll interact with:**

| File | What it contains | How to update |
|---|---|---|
| `data/raw/mapping.csv` | The main research data (threats, solutions, opportunities) | Replace with new CSV from researchers |
| `public/data/aspects.json` | The 16 democracy aspect definitions from the IDEA codebook | Update when the codebook changes (see Section 5) |
| `public/data/data.json` | Generated from the CSV — **never edit directly** | Auto-generated by preprocessing |

---

## Two ways to update data

### Option A: Admin Panel (easiest — no Git needed)

The website has a built-in admin panel for uploading new CSV data.

1. Go to `[your-site-url]/admin`
2. Log in with the admin credentials (ask the project maintainer)
3. Click **Upload CSV** and select your new `.csv` file
4. Review the preview — it shows:
   - How many threats, solutions, and opportunities were found
   - Any warnings (e.g., unknown aspect codes)
   - A sample of the first 5 items
5. If everything looks correct, click **Publish**
6. The old data is automatically backed up (you can restore it if needed)

**Limitations:** The admin panel only handles CSV data uploads. For codebook changes, README edits, or other file changes, use the Git workflow below.

---

### Option B: Git workflow (for all changes)

This is the full workflow for making any change to the project.

#### Step 1 — One-time setup

**Create a GitHub account:**
1. Go to [github.com](https://github.com) and click **Sign up**
2. Use your P4D email address
3. Choose a username (e.g., `aurora-p4d`)
4. Verify your email

**Install GitHub Desktop** (recommended for non-developers):
1. Go to [desktop.github.com](https://desktop.github.com) and download
2. Install and sign in with your GitHub account
3. The project maintainer will add you as a collaborator on the repository

**Clone the repository:**
1. In GitHub Desktop, click **File → Clone Repository**
2. Find the `aixd-threatmap` repository in the list (or paste the URL)
3. Choose a local folder (e.g., `Documents/aixd-threatmap`)
4. Click **Clone**

You now have a local copy of the project on your computer.

#### Step 2 — Make your changes

**To update the CSV data:**
1. Replace the file at `data/raw/mapping.csv` with your new CSV
2. The CSV should be semicolon-delimited (`;`) — this is the default when exporting from Excel in many European locales
3. The first row should be a title row, the second row empty, the third row should be column headers
4. Required columns: `Stable ID`, `Threat (paraphrased)`, `Threat (verbatim)`, `Solution (paraphrased)`, `Solution (verbatim)`, `Independent Opportunity (paraphrased)`, `Independent Opportunity (verbatim)`, `Source`, `P4Dem Category 1` through `P4Dem Category 6`

**To update the codebook** (aspects.json):
See Section 5 below.

#### Step 3 — Run preprocessing

This converts your CSV into the `data.json` file the website reads.

**If you have Bun installed:**
```bash
bun run preprocess
```

**If you have Python installed:**
```bash
python3 preprocessing/preprocess.py
```

You should see output like:
```
Read 201 data rows from data/raw/mapping.csv
Transformed 188 items (13 rows skipped)
  Threat+Solution: 103, Threat: 41, Opportunity: 44
Wrote public/data/data.json
OK: All 16 aspect codes are valid
```

If you see **WARNING** messages about unknown aspect codes, the CSV contains P4Dem codes that don't exist in `aspects.json`. Check the codebook.

#### Step 4 — Commit and push

**In GitHub Desktop:**
1. You'll see your changed files listed on the left
2. Write a short description of what you changed, e.g.:
   - "Update data with new CSV from March research round"
   - "Fix aspect code typo in row 45"
3. Click **Commit to master**
4. Click **Push origin** (the blue button at the top)

Your changes are now live. The website will rebuild automatically within a few minutes.

## Understanding the data pipeline

```
CSV file (data/raw/mapping.csv)
  ↓  preprocessing (Python script)
  ↓  validates aspect codes against aspects.json
  ↓  extracts types, citations, URLs
  ↓
data.json (public/data/data.json)  →  Website reads this
```

The preprocessing script:
- Skips rows where both threat and opportunity are empty (`///`)
- Determines the type: `threat-solution` (has threat + solution), `threat` (threat only), `independent-opportunity` (opportunity only)
- Extracts aspect codes from the `P4Dem Category` columns (e.g., "2.1 Free and Fair Elections" → code "2.1")
- Derives short citations (e.g., "George & Klaus (2026)") from full source strings
- Handles both comma-delimited and semicolon-delimited CSVs
- Handles both UTF-8 and Windows-1252 encoded files

## Updating the aspects codebook

The file `public/data/aspects.json` defines the 16 democracy aspects from the IDEA framework. Each aspect has:

```json
{
  "2.1": {
    "code": "2.1",
    "name": "Free and Fair Elections",
    "definition": "Effects on electoral integrity, voter access, campaign fairness, or trust in election outcomes.",
    "description": "Effects on the integrity of electoral processes, the inclusivity and accessibility of voting, the fairness of political competition, and public trust in elections as genuinely open contests.",
    "pillar": "Representative and Accountable Government",
    "pillarCode": "2"
  }
}
```

**If the codebook hasn't changed** — you don't need to do anything. The existing `aspects.json` is correct.

**If aspects have been added, renamed, or redefined:**

The easiest way is to use Claude (see next section). Open a Claude session, paste the new codebook text, and ask it to update `aspects.json`. The project has a skill file (`.claude/skills/update-aspects.md`) with detailed extraction rules.

Alternatively, edit `aspects.json` manually following these rules:
- The key must match the `code` field (e.g., key `"2.1"` → `"code": "2.1"`)
- `pillarCode` must match the integer part of the code (e.g., `"2.1"` → `"pillarCode": "2"`)
- All aspects in the same pillar must have the same `pillar` name
- `definition` is 1-2 sentences (short, for tooltips and badges)
- `description` is 1-3 sentences (longer, shown in the aspect detail dialog)

After editing, validate with:
```bash
python3 preprocessing/validate_aspects.py
```

You can use [Claude](https://claude.ai) or [Claude Code](https://claude.ai/code) to help with updates. Here's how:

## Troubleshooting

### "WARNING: Aspect code 'X.Y' found in data but not in aspects.json"
The CSV contains a P4Dem code that doesn't exist in the codebook. Either:
- The code is a typo in the CSV → fix the CSV
- The codebook has been updated with new codes → update `aspects.json` first

### Preprocessing says "0 items transformed"
- Check that the CSV has the correct column headers (row 3)
- Check that the delimiter is correct (semicolons or commas)
- Make sure rows have a `Stable ID` value

### "Push origin" is greyed out in GitHub Desktop
- You may not have push access. Ask the project maintainer to add you as a collaborator.
- Or you may need to "Fetch origin" first if someone else pushed changes.

### The website doesn't update after pushing
- The rebuild takes 2-5 minutes. Wait and refresh.
- Check the GitHub Actions tab on the repository for build errors.

### I accidentally broke something
- In GitHub Desktop, right-click the commit and select **Revert Changes**
- Or ask the project maintainer to restore from the admin panel's backup
